## 🧩 我的项目方法

| 阶段 | 我会关注什么 |
| --- | --- |
| 讲清问题 | 先明确场景、输入、输出和边界，而不是直接写代码 |
| 拆解流程 | 把任务拆成可观察的步骤，明确状态和中间结果 |
| 做出原型 | 先让系统可运行，再逐步补完整体体验 |
| 补齐文档 | README、运行方式、问题记录、下一步计划都要能被别人看懂 |
| 持续维护 | 保持少量真实提交，不为贡献图制造无意义内容 |
