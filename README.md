## 🚀 项目经历

### Multi-Agent 智能研究助手

围绕复杂问题研究场景，把一个问题拆成 **规划、搜索、分析、写作** 几个阶段。这个项目让我重点理解了：Agent 不只是调用模型，而是要设计状态、流程、工具输入输出和前端可观察性。

```text
关键词：LangGraph / Search Tool / WebSocket / React Flow / Structured Output
收获：理解多步骤 AI 应用如何落地成一个可观察的产品界面
```

### IDL / 领域文档 RAG 面板

围绕专业文档、脚本示例和代码资料，搭建检索增强问答面板。重点不是简单接入向量库，而是把 **全文检索、向量检索、融合排序和重排策略** 组合起来，让结果更可解释。

```text
关键词：FastAPI / SQLite FTS / LanceDB / RRF / SSE / React
收获：理解 RAG 系统的检索链路、降级策略和调试体验
```

### GIS / Web 可视化练习

基于地理信息方向的学习背景，持续整理空间数据、地图交互和 Web 可视化相关能力。这个方向是我和普通 Web 项目区分开的长期标签。

```text
关键词：PostGIS / WebGL / Web GIS / Spatial Data / Frontend Visualization
收获：探索 AI 应用与空间数据场景结合的工程方式
```

## 🧭 我如何做项目

我比较重视项目的后半段：文档、复盘、可运行性和后续维护。一个项目如果只能跑一次，但别人看不懂、复现不了，我会认为它还没有完成。

```text
先把问题讲清楚 → 再把流程拆明白 → 做出可运行原型 → 补文档、记录问题、持续维护
```

| 内容 | 状态 |
| --- | --- |
| 个人主页 | 正在调整成更像“对外介绍我”的主页 |
| 项目链接 | 暂时不直接放仓库列表，先补 README、截图、运行方式和隐私检查 |
| 学习记录 | 使用私有记录按周维护，保持真实提交，不刷贡献图 |
| 独立作品集 | 本地版本已搭建，后续可发布成更完整的网页作品集 |
